从原始 Level2 到分钟因子面板

因子生成前先把高频数据降噪成可对齐、可滚动、可回测的分钟级表，尽量保留盘口与成交的微观结构信息。

1. Market Snapshot

3 秒级快照提供 10 档 bid/ask 的价格与挂单量。分钟表主要采样前 5 档，并派生 spread、mid price、盘口深度、买卖盘不平衡与盘口重心。

重点字段bid_px_i、ask_px_i、bid_vol_i、ask_vol_i

2. Orderbook

订单簿用于刻画挂单新增、撤单、队列变化和深度稳定性。它给 LLM 提供“流动性供给侧”的语言空间，例如共识价格是否稳定、流动性是否突然抽离。

研究方向queue pressure、depth slope、anchor stability、cancel intensity

3. Tradebook

成交簿用于刻画主动成交、放量下跌、冲击成本和成交价相对均价的偏离。与订单簿结合后，可以区分“价格跌但流动性稳”和“价格跌且盘口失稳”。

研究方向volume weighted return、trade imbalance、price impact、VWAP deviation

LLM 不是直接“猜一个因子”，而是被放进固定契约：读 schema、写方向、写逻辑、写代码、跑样例、回溯评估，失败样本会回流到知识库。

评估纪律：不穿越、不交叉、看滚动

订单簿因子很容易因为日内聚合和历史窗口写错而引入未来信息，因此评估层把时序边界作为第一优先级。

时序切分

  • 严格采用历史因子值和未来收益标签，训练、验证、观察窗口不交叉。
  • Rank IC、分组收益和覆盖率按时间滚动观察，方便后续滚动切训练集。
  • 单日内订单簿特征只用当日当前 bar 之前的信息，不使用收盘后才能知道的状态。

Rolling 约束

  • 日 bar 历史特征采用单边 rolling，避免在标准化、均值回归或波动率估计中混入未来。
  • 分钟级窗口明确 lookback 和 shift，生成代码后用样例数据做接口和边界测试。
  • 对极端缺失、停牌、无成交和盘口为空的资产单独标记，防止异常值伪装成 alpha。

双层去重：语义相似 + 数值相关

LLM 会天然生成许多“换个说法但本质相同”的因子，因此先用知识库做语义去重，再用截面和时序相关性做数值去重。

语义去重

把因子方向、字段依赖、窗口、聚合方式和经济解释写入知识库。新因子入库前先检索相似描述，避免重复生成“深度不平衡”“买卖盘压力”“价差扩张”等同义模板。

保留原则经济含义更清晰、字段依赖更少、历史稳定性更好的版本优先。
淘汰原则仅窗口长度、字段命名或正负号不同，但解释与表现高度重合的版本剔除。

两个高 Rank IC 因子例子

下面不是最终投资建议，而是展示 Agent 如何把订单簿/成交簿现象翻译成可解释、可回测的因子代码逻辑。

trade_topneg20_ret_ratio

放量下跌的相对抗跌能力

如果该因子负得很大，说明最近 20 根 K 线中“放量下跌”更严重；如果负得较小，则代表在下跌成交中表现更抗跌，可能体现相对强度。

r_t = close_t / close_{t-1} - 1 w_t = volume_t / total_volume factor = Σ(r_t × w_t), t ∈ 前20根K线 且 r_t < 0
std(depth_anchor_stability_ratio)

订单簿共识不稳定性

括号内衡量成交价或均价与订单簿重心价格的偏离程度；其标准差越高，说明成交与盘口共识之间越不稳定。该因子 Rank IC 为负，bottom 组超额更明显，可命名为 orderbook_consensus_instability。

anchor = weighted_mid_price_from_depth ratio_t = abs(trade_price_t / avg_price_t - anchor_t / avg_price_t) factor = std(ratio_t over lookback_window)

产出物与后续迭代

最终沉淀的不是单个公式，而是一套可持续生产、审计和复用的因子工厂。

因子资产

保留因子代码、依赖字段、参数、解释文本、样例测试结果、回溯区间、Rank IC 和分组收益，形成可检索的研究资产。

失败样本

把无法运行、超时、未来信息风险、样本覆盖过低或解释不清的因子归档，作为下一轮 prompt 约束和反例。

滚动训练

候选池按相关性和稳定性筛选后进入滚动训练集，后续可以继续做行业中性、风格暴露控制与组合层容量评估。
